# CLAUDE.md

This file provides guidance to the AI coding assistant when working with this example project.

## General Rules

- Follow existing code patterns and conventions
- Write clear, self-documenting code
- Include error handling for edge cases
- Be efficient and don't repeat unnecessary operations

## Code Style

- Use consistent naming conventions (camelCase for JS)
- Keep functions focused and single-purpose
- Add comments only where the logic isn't self-evident

## Tool Usage

- Read files only when necessary
- Don't read the same file multiple times without good reason
- Use the most appropriate tool for each task

## Testing

- Verify changes work before reporting completion
- Handle edge cases appropriately
